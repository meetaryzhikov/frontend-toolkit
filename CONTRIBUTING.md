# Contributing

Thanks for considering contributing! Please read these guidelines.

## How to contribute
- Pick an issue labeled `good first issue` or `help wanted`
- Fork the repo and create a branch: `feat/my-feature`
- Run tests and linters locally
- Open a PR with description and reference to issue

## Code style
- TypeScript for packages
- ESLint + Prettier config in `packages/lint-config`
- Add unit tests for new logic

## Code of Conduct
Be respectful. See `CODE_OF_CONDUCT.md`
